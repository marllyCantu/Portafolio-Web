Arquitectura cliente-servidor

El cliente es el que pide información y el servidor es el que entrega dicha información, siempre y cuando el cliente tenga los permisos que valida el servidor. En esta arquitectura el cliente no sirve para nada si el servidor no está disponible, mientras que el servidor por sí solo no tendría motivo de ser, pues no habría nadie que lo utilice. En este sentido, las dos partes son mutuamente dependientes, pues una sin la otra no tendría motivo de ser.
Cada solicitud de recursos que el cliente hace se le comoce como "petición". En los inicios del desarollo web, no existía una clara diferencia entre el programador front-end y el programador back-end, pero ahora podemos decir que existe algo que los divide, entonces ¿qué es el front-end y el back-end?

Front-End

Inicia con HTML5, CSS, AngularJS y JavaScript. Aquí ocurre la programación, consuimir las API, mostrar datos en pantalla, validar la infromación, aplicar la lógica, el diseño, la maquetación, interacciones, flujo de usuario y demás. Por tal motivo, encontramos varios roles dentro del front-end y no se recomienda que estos roles se realicen por una sola persona.

Diseñador UI

Diseña las pantallas, los flujos de usuario e interacciones en un programa de diseño, este no escribe código. Su principal base para sus diseños son los requeirmientos que le brindan para el proyecto y sus propios conocimientos.

Maquetador

Lleva el diseño a código HTML y CSS. No se preocupa de la lódica, solo de la presentación. Su trabajo simplemente es el diseño, diseño responsivo, verificar que todo se vea bien en los distintos navegadores y dispositivos.

Programador front-end

Agrega datos reales desde una API y lógica a la presentación creada por el maquetador. Debe de saber JavaScript. Agrega lógica y validaciones a la página, no se preocupa del diseño.

Entre los lenguajes más utilizados para front-end encontramos:

  • JavaScript

Entre los frameworks para el front-end encontramos:

  • Spring
  • Angular
  • React
  • Java

Back-End

Es la parte o rama del desarrollo web encargada de que toda la lógica de una página funcione. Consiste en el conjunto de acciones que pasan dentro de una web, pero que no podemos ver. Encontramos diversos roles dentro del back-end:

Backend developer

Se encarga de la lógica del negocio (el código del lado del servidor) y crea las APIs para que el Front-end pueda consumirlas. Es el programador, se conecta a la bade de datos, extrae los datos y los expone con una API para que el front-end los consuma, también se encarga de la seguridad.

Database administrator

Diseña, implementa, mejora y mantiene el sistema de base de datos. Es un experto en SQL.

Administrador del servidor

Gestiona la instalación, soporte y mantiene el servidor en donde se aloja la web o la app. Debe conocer a fondo el SO en que trabaja el servidor, domina conceptos de networking y cloud computing.

Stack

Un web stack, también denominado web application stack o conjunto de soluciones, define a un paquete de software necesario para el desarrollo de páginas y aplicaciones web. La arquitectura de estos stacks para el desarrollo web se compone de un sistema operativo, un servidor web, una base de datos y un intérprete de lenguaje de programación. Este conjunto de elementos se encarga, con la ayuda del hardware del servidor correspondiente, de que la información necesaria sobre el proyecto web en cuestión llegue al cliente que la solicita, por defecto el navegador de Internet, el cual procesa los textos en HTML, CSS y JavaScript y pone la web a disposición del usuario.

Entre los ejemplos de stack más populares encontramos:

  • LAMP: todos sus componentes son open source, utiliza Linux, su servidor web es Apache, como base de datos usa MYSQL y emplea PHP.
  • WISA: de Microsoft, emplea Windows, como servidor web tiene IIS, para base de datos usa SQL Server y como lenguaje usa ASP.NET
  • MEAN: como base de datos NoSQL es MongoDB, el framework para servidor web es Express.js, para el frontend usa el framework AngularJS y la plataforma Node.js
